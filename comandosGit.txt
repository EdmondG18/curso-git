- git -v: Para ver la version de git descargada.

- git --help: Para ver los comandos de git disponibles.

- git config --global: Sirve para configurar caracteristicas globales del git.
  - Si agregas user.name "name", se agrega a la configuracion el nombre del usuario.
  - Si agregas user.email "email", se agrega a la configuracion el email del usuario.

- git init: Para inicializar un repositorio en la carpeta en la que estas.

- git status: Para ver el estado del repositorio. 

- git add: Para trackear un archivo que tiene modificaciones hechas.
  - git add . : Se trackean todos los archivos que tienen modificaciones.
  - git add <file> : Se trackea el archivo que se menciona.

- git restore --staged <file>: Para anular el git add y que vuelva al estado anterior.

- git commit -m "Message": Se hace commit y se pone el mensaje de lo que se hizo en esos cambios. 

- git log: Muestra todos los commits hechos, especificando el nombre y email del usuario que los hizo.
  - git log --graph: Para ver los commits como si fuera una rama.
  - git log --graph --pretty=oneline: Para ver los commits como si fuera una rama, y todo en una linea.

- git checkout: Para descartar todos los cambios hechos antes de hacer add (actualmente es mejor: git restore archivo). Tambien para moverse entre ramas (acualmente es mejor: git switch rama). Solo puedo ver o restaurar archivos. No puedo eliminar commits. POCO PRACTICO EN LA ACTUALIDAD

- git reset: Para descartar todos los cambios hechos con git add.
  - git reset --hard [nro commit]: Dependiendo el numero de commit se regresa a ese commit y elimina todos los anteriores del registro.
  - git reset --soft [nro commit]: Vuelve al numero de commit especificado pero deja los cambios hechos para hacer commit de nuevo.

- git switch rama/commit: Para viajar entre ramas o entre commits. Si viajo a un commit antiguo (con el codigo) no elimino los antiguos si no que cambio el puntero de la cabeza de la rama en la que estoy para inspeccionar codigo de los commits anteriores.
  - git switch master: Vuelvo a la rama master y a donde esta el ultimo commit hecho.

- git config --global alias.name "long name": Para crear un alias a un comando que sea mas largo y guardarlo.

- git diff: Para ver los cambios hechos respecto al ultimo commit.

- git reflog: Para ver el historial de todo lo que he hecho en git.

- git tag: Para etiquetar el Commit en donde me encuentro. Como agregarle un nombre al nodo.

- git branch: Sirve para ver todas las ramas que tengo.
  - git branch name: Sirve para crear una rama nueva.

- git merge: Sirve para combinar los datos de ramas para mantener el control.

- git stash: Para dejar una rama en pausa con datos guardados sin hacer commit y que me deje cambiar de rama sin ese commit.
  - git stash list: Para ver la lista de los stash guardados.
  - git stash pop: Para que cuando vuelva a la rama recuperar los datos del stash.
  - git stash drop: Para eliminar todo lo que haya en el stash antes de recuperarlo, porque no lo queremos al final.
